# InstaBOT on Render Free Tier — Fixed & Production-Ready

## What this fixes

The previous guide exists but has gaps. This guide guarantees the bot works on Render Free
by solving every known blocker:

| Blocker | Risk | Fix |
|---------|------|-----|
| Ephemeral filesystem | `account.txt` + SQLite wiped on redeploy | Env vars + MongoDB Atlas |
| Sleep after 15 min idle | Cold start delays, bot goes offline | Uptime monitor pings every 5 min |
| Free Postgres expires in 90 days | Database gone after 90 days | MongoDB Atlas (free forever) |
| No outbound SMTP | Email notifications blocked | Telegram/Discord webhooks |
| Cold start health check failure | Render marks instance unhealthy | Proper `/health` endpoint + ping |
| GitHub Actions deploy hook | Secrets exposed if misconfigured | Use Render Variables (not secrets) |

## Step 1 — Fix the cookie storage (critical)

Render wipes `account.txt` on every redeploy. The fix: store cookies as an
environment variable instead of a file.

### 1a. Export your current cookie

In Termux, get your cookie string from `account.txt`:

```bash
cat ~/InstaBOT/account.txt
```

Copy the entire output (it's a Netscape cookie file format). You'll paste this
as a single-line value in the next step.

### 1b. Update `config/default.json`

```json
"instagramAccount": {
  "email":    "",
  "password": "",
  "2FASecret": "",
  "i_user":   "",
  "proxy":    null,
  "userAgent": "Mozilla/5.0 (Linux; Android 14; SM-S928B) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/131.0.0.0 Mobile Safari/537.36",
  "intervalGetNewCookie": 1440
},
```

### 1c. Update `config/index.js`

Find the `ACCOUNT_FILE` line and update it to support env var:

```js
// Around line 161 — change this:
ACCOUNT_FILE:  process.env.ACCOUNT_FILE || './account.txt',

// To this (supports inline cookie via env var):
ACCOUNT_FILE: process.env.ACCOUNT_COOKIE
  ? '/tmp/account.txt'   // write env var to temp file on startup
  : (process.env.ACCOUNT_FILE || './account.txt'),
```

Then add startup code in `bot/InstagramBot.js` to write the env var to a temp file:

```js
// At the top of start() or loadAndLogin(), add:
if (process.env.ACCOUNT_COOKIE) {
  const fs = require('fs');
  fs.writeFileSync('/tmp/account.txt', process.env.ACCOUNT_COOKIE, 'utf-8');
}
```

### 1d. In Render dashboard, set this env var

```
ACCOUNT_COOKIE=# Netscape HTTP Cookie File
# http://curl.haxx.se/rfc/cookie_spec.html
# This file was generated by Cookie-Editor
HttpOnly_.instagram.com TRUE / TRUE 1808152822 ps_n 1
HttpOnly_.instagram.com TRUE / TRUE 1806742851 datr Q12haRJEdtEX0BHCBEn4tnCI
...(paste full content here, all on one line)
```

**Important**: The value must be on ONE line. Use `\n` for newlines if needed,
or paste the raw multi-line text — Render supports multi-line env var values.

## Step 2 — Fix the database (critical)

SQLite writes to disk. On Render Free, the disk is wiped on every redeploy.
The solution: **MongoDB Atlas** (free forever, no credit card).

### 2a. Create MongoDB Atlas cluster (free)

1. Go to [mongodb.com/atlas](https://www.mongodb.com/atlas)
2. Sign up → Create Free Cluster (M0 Sandbox)
3. Region: Singapore or closest to your users
4. Click **Database Access** → Add New User:
   - Username: `instabot`
   - Password: (generate secure password, save it)
5. Click **Network Access** → Add IP Entry → Allow Access from Anywhere: `0.0.0.0/0`
6. Click **Clusters** → **Connect** → **Connect your application**
7. Copy the connection string:

```
mongodb+srv://instabot:YOUR_PASSWORD@cluster0.xxxxx.mongodb.net/instagram_bot
```

### 2b. Update `config/default.json`

```json
"database": {
  "type": "mongodb",
  "uriMongodb": "",
  "autoSave": true,
  "saveIntervalMinutes": 1,
  "autoSyncWhenStart": false,
  "autoRefreshThreadInfoFirstTime": true
},
```

### 2c. In Render dashboard, set these env vars

```
MONGODB_URI=mongodb+srv://instabot:YOUR_PASSWORD@cluster0.xxxxx.mongodb.net/instagram_bot
DATABASE_TYPE=mongodb
```

The `config/index.js` already reads `MONGODB_URI` and `DATABASE_TYPE` from
`process.env`, so no code changes needed beyond step 2b.

## Step 7 — Set up uptime monitoring (prevent spin-down)

Render Free spins down after 15 minutes of no traffic. Without this, your bot
will be dead the next time Instagram sends it a message.

### Option A: cron-job.org (completely free, recommended)

1. Go to [cron-job.org](https://cron-job.org) — sign up free
2. Click **Create cron job**
3. URL: `https://your-service.onrender.com/health`
4. Schedule: `Every 5 minutes`
5. Timezone: `Asia/Dhaka`
6. Save

This will ping your bot every 5 minutes, keeping it alive 24/7.

### Option B: UptimeRobot (50 free monitors)

1. Go to [uptimerobot.com](https://uptimerobot.com) — sign up free
2. Click **Add New Monitor**
3. Type: HTTP(s)
4. Friendly name: `InstaBOT Render`
5. URL: `https://your-service.onrender.com/health`
6. Monitoring interval: `5 minutes`
7. Save

## Step 8 — Fix GitHub Actions auto-deploy (optional)

Update `.github/workflows/deploy.yml` to use Render Variables instead of Secrets:

```yaml
name: Deploy to Render

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Trigger Render Deploy
        run: |
          curl -s -X POST "${{ vars.RENDER_DEPLOY_HOOK_URL }}" \
            -H "Content-Type: application/json"
```

In Render dashboard → your service → **Deploys** → **Automate Deploys** →
toggle ON → copy the webhook URL.

In GitHub repo → **Settings** → **Variables** (not Secrets) → **New repository variable**:
- Name: `RENDER_DEPLOY_HOOK_URL`
- Value: (paste the webhook URL from Render)

Using Variables instead of Secrets is safer — Variables are not exposed in logs.

## Troubleshooting

### "Connection refused" on health check
The bot takes 30-60 seconds to start. Wait 2 minutes after deploy before testing.

### MongoDB connection failed
- Check `MONGODB_URI` is correct (no `<>` around your password)
- Check Atlas Network Access allows `0.0.0.0/0`
- Check Atlas cluster status is "Online" (not "Paused")

### Bot sleeps despite uptime monitor
- cron-job.org might be blocked by Render. Use UptimeRobot instead.
- Check the ping URL is exactly `https://your-service.onrender.com/health`
- Make sure the ping monitor has been active for at least 15 minutes

### Cookie expired / login failed
Instagram cookies expire. Set `intervalGetNewCookie: 1440` in your config
(it already is). The bot will re-login using email/password from env vars.
Make sure `ACCOUNT_EMAIL` and `ACCOUNT_PASSWORD` are also set in Render.

### "Ephemeral filesystem" errors in logs
- This is normal on Render Free. As long as `ACCOUNT_COOKIE` env var is set
  and MongoDB Atlas is connected, the bot will recover on restart.

### Instance hours exceeded (750h/month)
- Free tier gives 750 hours. One running instance = 744 hours/month.
- If you run multiple services or exceed this, Render suspends (not bills).
- Upgrade to Starter $7/mo only if needed.
